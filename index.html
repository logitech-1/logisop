Logitech Sync Portal 完整操作手冊

針對無須雲端連線，僅需使用報修與授權功能之企業用戶

💡 核心觀念： Sync Portal 採用安全雲端架構。但如果您不想將設備連線至 Sync，也完全沒問題！使用 Sync 來「開立客服工單」或「管理合約授權」，並不需要將實體設備連線。

★ 從 Service Portal 移轉至 Sync 的三步驟

如果您以前有在使用舊版的 Select Portal，登入時系統會引導您完成帳號與設備的轉移：

第一階段：發起移轉 (Initiation)

  • 觸發提示：在舊版 Select Portal 頂端會看到 Banner 提醒，請點擊 Start the process (開始移轉)。
  • 決定轉移去向 (三選一)：
    • 選項 A (Aneez Ventures)： 直接從清單中勾選您要接收舊設備的 Sync 組織。
    • 選項 B (找不到組織)： 點選 My Sync Organization is not listed，並手動輸入您的 Sync Organization ID。
    • 選項 C (需拆分多個組織)： 若要將設備分配到不同的 Sync 組織，請勾選 Move manually added rooms... to multiple Sync Organizations，此時系統會提示您發送 Email 至 select@logitech.com，由羅技服務團隊人工協助處理。
  • 最終確認 (不可逆)：點擊 Next 後會跳出警告視窗。確認無誤後點擊 Okay 正式啟動。此過程一旦確認便無法復原。

⚠️ 補充說明：哪些東西「不會」被自動移轉？
1. 純個人周邊設備： 像是 Wave Keys 鍵盤、MX Vertical 滑鼠、C615 視訊鏡頭等非會議室系統的周邊無法轉移（註：會議室用的 Rally Mic Pod 會被歸類在 Room System 內一併轉移）。
2. 舊版 Portal 的使用者： 因為兩邊系統權限設計不同加上隱私考量，舊系統的同事帳號不會自動搬過去。請依照下方第 2 章節的步驟，在 Sync 重新發 Email 邀請同事。

1. 帳號開通與加入組織

1.1 註冊流程

  • 前往註冊：進入 Sync Portal，點擊 Sign up。
  • 填寫資料與驗證：輸入商務 Email 與公司名稱。同意隱私條款並過完 reCAPTCHA 驗證後，去信箱點擊 Verify My Email，完成後續資料填寫。

1.2 常見的加入情境 (Request Access vs. Create New)

當您輸入公司 Email 時，系統會自動偵測您的網域：

  • 出現 Request access：代表有同事已經建立過同網域的組織。我們強烈建議點擊 Request access 申請加入，避免在系統內產生重複的組織。您的申請會送到管理員那裡審核（View Requests）。
  • 只出現 Create a new organization：代表您輸入的組織名稱已被別人註冊，請換一個名稱（例如加上分公司名稱）再試一次。
⚠️ 擁有者限制：同一個 Email 只能作為「Owner」建立一個租戶。若貴公司需要切割多個獨立的 Sync 租戶，請使用 Email 別名 (Alias) 建立新租戶，再將您的主要 Email 加進去當管理員。

2. 使用者管理 (Users)

新增使用者需透過 Email 邀請。請到左側選單的 System > Users 進行管理。

2.1 完整權限角色說明 (User Roles)

角色名稱 權限說明
Owner (擁有者) 最高權限。擁有平台所有功能。可以檢視與分配合約授權 (License)。
3rd Party (第三方) 專為外部維護廠商設計。可增刪房間、設備，且可以分配授權 (License)。可開工單。
Sync Admin (管理員) 可增刪一般使用者、管理房間與終端設備。可開工單。(無法管理授權)
Device Admin (設備管理) 可增刪/修改設備，但無法管理使用者或群組。可開工單。(無法管理授權)
Device Manager 僅能「管理」設備（無法新增/刪除設備，也不能動使用者與授權）。
Installer (安裝員) 可新增設備/房間、移動設備位置、刪除房間。無法管人與授權。
Support Only 純客服聯絡人。只能看，以及開立/追蹤報修工單。不能改任何設定。
Read Only (唯讀) 只能檢視，不能做任何更改，也不能開立工單。

2.2 進階帳號管理操作

  • 編輯權限：在使用者列表中點選該使用者，右側會滑出面板，您可以更改其 Role（角色），或限制他們只能看到特定群組的資料 (Access Control)。注意：Email 一旦建立無法更改，只能刪除重建。
  • 下載名單：點擊列表上方的 Download Data，可以將所有使用者名單匯出為 .csv 格式（包含建立時間與更新時間）。
  • 刪除使用者：只有 Owner 或 3rd Party 可以刪除其他使用者。勾選使用者後，點擊 Delete 並確認即可。

3. 服務授權與合約管理 (Contracts)

購買羅技服務（如 Essential 或 Select）後，請確保只有 Owner 或 3rd Party 權限的帳號來進行操作。

3.1 匯入與查看合約

  • 點擊左側選單的 Contracts。這裡會集中顯示您所有的合約。
  • 下載 PDF：點擊特定合約，可以查看剩餘效期與授權數量，畫面中也可以直接下載該合約的 PDF 檔。
  • 修改名稱：點擊合約號碼旁邊的「編輯 ✏️」圖示，可以為合約取一個好記的自訂名稱（例如：台北總部 2024）。原本的合約號碼依然會保留顯示。

3.2 三種分配授權 (Assign License) 的方法

  • 方法 A：批次分配 (Bulk selection) 進入 Inventory，把需要授權的房間全部打勾，點選上方工具列的 Assign License，接著在跳出的視窗中選擇對應的合約即可。
  • 方法 B：單一設備分配 在 Inventory 列表直接針對單一房間進行授權綁定。
  • 方法 C：從房間內部配置 點擊進入某個特定的 Meeting Room，在上方導覽列切換到 Details 頁籤。在下方會看到 Contract 的下拉選單，直接挑選合約即可。

4. 客服工單報修 (Support Ticketing)

4.1 兩種開立工單的途徑

  • 途徑一：從 Support Tickets 總表開單
    點擊左側 Support tickets (救生圈圖示) > 點擊右上角 Create a ticket > 手動選擇有問題的產品並填寫資訊。
  • 途徑二：直接從故障的房間開單 (最推薦、最快)
    到左側 Meeting Rooms/Desks > Inventory 找到出問題的房間 > 切換到 Support 頁籤 > 點擊 Create a ticket。系統會自動帶入該房間的設備與綁定的授權等級，直接填寫故障情況即可。

4.2 工單狀態說明 (Ticket Status)

狀態 說明
Open 工單已建立，Logitech 客服團隊正在檢視或處理中。
Awaiting your reply 客服已回覆，正在等待您提供進一步資訊（需要您的動作）。
Solved 問題已解決（或因閒置多日被系統標記為解決）。您可以在 7 天內回覆以重新開啟它。
Closed 已徹底關閉的工單。無法重新開啟，若有後續問題需開立新工單。

4.3 加入追蹤者 (Sync Followers)

您可以把同事加入工單的 Followers 讓他們一起追蹤進度。請注意：外部信箱（沒有 Sync 帳號的人）也能被加入追蹤，但他們只會收到 Email 通知，無法登入 Sync Portal 看工單。

4.4 歷史舊工單轉移與分享 (重要！)

如果您以前在舊版 Business Support 網站開過工單，請留意以下規則：

  • 預設是隱私的：舊網站來的工單，剛轉移過來時只有您自己看得到。
  • 如何分享給全公司：進入 Support > Business Support，點擊該舊工單，點擊右上角的 MANAGE TICKET，選擇 Share。分享後，全組織都能在 All spaces 看到它。
⚠️ 不可逆操作：分享舊工單的動作是無法復原的 (cannot be undone)。如果您身兼多個 Sync 組織，請確認您正在正確的組織內進行分享。

4.5 常見問答：如果同事離職了，他的工單會怎樣？

  • 他在 Sync 裡面建立的工單，會永遠保留在組織內。
  • 他以前在舊系統開的工單，如果已經按過 Share，團隊依然可以檢視與編輯。
  • 他以前在舊系統開的工單，如果從未分享過，那該工單會跟著他的 Email 帶走，組織將無法查看。
